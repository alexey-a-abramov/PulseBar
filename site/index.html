PulseBar

An advanced, interactive Touch Bar for macOS.

A native menu-bar agent that takes over the Touch Bar — regardless of which app is focused — and turns it into a live system monitor and a control surface you can actually press.

The public NSTouchBar API only shows up when your app is frontmost. PulseBar instead presents a system-modal bar that stays put across every app, updating once a second — so your machine's vitals and your favourite controls are always one glance (and one tap) away, no matter what you're doing. And it does it as a true background citizen: built and tuned for Apple Silicon, it sips ~2% CPU and ~47 MB of RAM on an M1.

Tiles that do something

Every metric tile is live; tap one to cycle its view (sparkline ↔ fundamentals). Controls respond to taps and drags in real time.

Any city from a master list, DST/summer-time correct, with a live ±offset and a next/prev-day badge. Add several.

⚡

GPU · NET · DISK

GPU utilisation, live ↓/↑ throughput, and read/write I/O with free space — each with its own sparkline.

🎛️

Controls

Now-playing transport, volume & brightness sliders, Pomodoro, screenshot, lock — drag and tap, right on the bar.

Composable & ✕-aware

Layout profiles

macOS sometimes shows a close box (✕) that shoves the bar right. Every profile reserves room for it so the agent orb on the right is always visible. Switch with one tap from the menu — or fine-tune any tile in the editor.

Default rich

Auto density and the full mode-tab accordion — the everyday layout, applied on first run.

Minimum roomy

The same ✕-aware margins, but icon-only tiles and a collapsed tab strip (active pill + ›) to free maximum width for content.

Widgets are composable per mode: add a world clock, an app launcher, or any tile to any mode via the in-app editor — the same tile can live in several modes. A size-aware packing engine caches its work, so none of it costs a frame.

How it owns the bar

The honest part

To hold the bar persistently across apps, PulseBar uses the same private SPI as Pock / MTMR / BetterTouchTool — all verified at runtime on the build machine before use: DFRFoundation for focus-independent presentation (placement:1 hides the Control Strip natively), plus DisplayServices, MediaRemote and CoreAudio for brightness, now-playing and volume.

⚠️ Private API → not App-Store-shippable; Touch Bar Macs only. Built & tested on a MacBook Pro (M1 13″, macOS 15.6). Guarded so it degrades to a menu-bar item where the SPI is missing.

Get it running

Build · Run · Test

No Xcode project to wrangle — three shell scripts. Runs as a background accessory app (no Dock icon; a ▦ menu-bar icon).

# build → build/PulseBar.app (ad-hoc signed) + bare binary
./build.sh

# build-if-needed, then launch — look at your Touch Bar
./run.command

# unit tests (all samplers) + smoke test (presents the bar, clean exit)
./tests/run_tests.sh

Quit any time via the ▦ menu-bar icon → Quit PulseBar. Open Settings from the same menu to tune layout profiles, shortcuts, per-app auto-switch, the on-device model, and more.
